# Style System

## Purpose

Use this file when the user asks for 문체 통일, 문장 호흡, 가독성, 서술 방식, 내면 독백, or overall prose polish.

## Baseline Rhythm

- Default paragraph: 2-4 complete sentences.
- Default sentence: clear subject/action/result flow.
- Default webnovel scene rhythm: pressure -> action -> sensory/status detail -> short interpretation -> next pressure.
- Use short standalone lines only for dialogue, hard turns, emotional buttons, system/status blocks, sound effects, and episode-ending hooks.

## Reference-Reading Findings

Strong reference chapters rarely win by using fragmentary lines everywhere. They usually open with a strong attitude, thesis, ordinary pressure, system trigger, social insult, or sensory danger, then keep the paragraph unit readable while letting the scene move.

The common rhythm is:

1. State a biased view of the world or a concrete pressure.
2. Show the protagonist's personal cost.
3. Introduce the genre trigger quickly.
4. Let action or dialogue test the rule.
5. Attach a small reward, cost, or public reaction.

Use this as the default structure when a draft feels vague or too literary.

## Complete-Sentence Paragraph Rule

Avoid fragment stacks for ordinary narration.

Weak:

```text
왼쪽.
벽 표시 두 개.
오른쪽.
낮은 천장.
```

Stronger:

```text
나는 분필로 남겨둔 표시를 따라 왼쪽으로 꺾었다. 벽에 그어둔 선 두 개가 보이면 다음은 오른쪽이었다. 그 뒤로는 천장이 낮아져 자연스럽게 고개가 숙여졌다.
```

Use this rule for movement, inventory checks, visual observations, preparation, and route tracking.

## Density Control

- Light Naver style: shorter sentences, more dialogue/reaction, less explanation before action.
- Dark growth style: one sharp sensory detail, short self-interpretation, visible survival pressure.
- Cider/reversal style: insult -> calm response -> visible proof -> public reaction -> opponent panic.
- Action style: setup -> movement -> impact -> consequence -> reaction/new threat.
- Exposition style: test the rule under danger, show the cost, then name the rule.

## Line Function Rule

Every short line must have a job:

- button: emotional snap, joke, dread, decision;
- status: system or reward block;
- impact: sound or hit;
- transition: scene cut or time marker;
- hook: final next-click pressure.

If a short line is only an observation, direction, object, or preparation item, fold it into a complete sentence paragraph.

## Internal Monologue

Internal monologue should do one of these jobs:

- reveal fear, shame, calculation, desire, or denial;
- convert information into a choice;
- show the protagonist's wound or contradiction;
- sharpen a joke, suspicion, or dread.

Cut internal monologue when it only repeats what the scene already shows.

## Style Unification Protocol

When unifying several chapters:

1. Pick the dominant tone in one sentence.
2. Keep paragraph length consistent.
3. Normalize notation before polishing prose.
4. Make each recurring character's speech shorter and more recognizable.
5. Replace repeated poetic abstractions with concrete pressure, action, or sensory detail.
6. Keep one or two signature emotional lines per chapter, not one every paragraph.

## Common Failure Fixes

- Too many line breaks: merge related action/observation into complete paragraphs.
- Too bookish: move explanation after dialogue/action and make speech plainer.
- Too flat: add visible consequence, social reaction, or body feedback.
- Too vague: replace mood with a concrete object, number, name, deadline, or threat.
- Too repetitive: keep the strongest emotional sentence and cut the rest.
