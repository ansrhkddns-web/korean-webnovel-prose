# Revision Rubric

## Purpose

Use this file for diagnosing and rewriting drafts that feel inconsistent, too literary, too fragmented, too slow, too school-heavy, too exposition-heavy, or weak in dialogue.

## Diagnosis Order

1. Reader promise: what pleasure did the chapter promise?
2. Opening pressure: does the first paragraph start the engine?
3. Protagonist action: does the lead choose or only react?
4. Reward: what visible payoff appears?
5. Dialogue: does each exchange move pressure or status?
6. Description: does detail serve action, danger, reward, or emotion?
7. Exposition: is explanation delayed until the reader wants it?
8. Ending hook: is the final pressure concrete?
9. Notation: are status, memo, chat, and system formats consistent?

## Rewrite Levels

### Level 1: Cleanup

Fix notation, line breaks, repeated wording, and awkward sentences while preserving scene structure.

### Level 2: Prose Upgrade

Rebuild paragraph rhythm, dialogue, description, and emotional beats while preserving event order.

### Level 3: Scene Repair

Move reveals, add pressure, add reward, cut explanation, or change scene order.

### Level 4: Episode Rebuild

Redesign the chapter function, reward, ending hook, and continuity handoff.

## Fragment Repair

When a draft uses too many short fragment lines:

1. Group related observations.
2. Add subject and verb.
3. Connect route/action/result.
4. Keep only the strongest emotional or hook line standalone.

## Dialogue Repair

When dialogue feels bookish:

1. Shorten the line.
2. Make motive implicit.
3. Add interruption, dodge, or status pressure.
4. Replace explanation with a concrete accusation, offer, threat, or question.

## Exposition Repair

When explanation comes too early:

1. Start with a problem or failed attempt.
2. Let the protagonist test or pay for the rule.
3. Name only the part of the rule needed now.
4. Leave one unanswered piece for later.

## Final Pass Checklist

- One chapter, one primary pleasure.
- One scene, one dominant pressure.
- One paragraph, one clear flow.
- One dialogue exchange, one shift.
- One description axis, one function.
- One ending, one concrete next-click question.
